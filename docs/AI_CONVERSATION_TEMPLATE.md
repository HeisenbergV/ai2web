每次开启新会话时，可以复制以下模板并根据需求修改，让AI快速理解你的需求。

## 🎯 标准对话模板

### 模板 1：添加新功能页面

```
你好！我需要添加一个新功能页面。

【功能描述】
- 功能名称：用户管理
- 页面路径：/users
- 主要功能：显示用户列表，支持分页、搜索、添加、编辑、删除

【需求详情】
1. 列表页面：显示用户ID、姓名、邮箱、创建时间
2. 搜索功能：支持按姓名和邮箱搜索
3. 操作功能：添加用户、编辑用户、删除用户
4. 分页功能：每页10条，支持切换每页条数

【参考】
- 可以参考现有的 TableDemo 页面
- 使用 CommonTable 组件
- API 接口：GET /api/users（列表）、POST /api/users（添加）、PUT /api/users/:id（编辑）、DELETE /api/users/:id（删除）

请帮我完成这个功能。
```

```

```

## 💡 对话技巧

### 1. 明确需求

**好的描述：**
```
我需要添加一个用户管理页面，包含：
- 用户列表（分页）
- 搜索功能（按姓名）
- 添加用户功能
- 编辑用户功能
- 删除用户功能
```

**不好的描述：**
```
做个用户管理
```

### 2. 提供上下文

**好的描述：**
```
参考 src/views/TableDemo.vue 的实现方式，
创建一个类似的订单管理页面。
```

**不好的描述：**
```
做个订单管理
```

### 3. 明确文件位置

**好的描述：**
```
在 src/views/UserList.vue 文件中，
添加一个"状态"列。
```

**不好的描述：**
```
加个状态列
```

### 4. 提供 API 信息

**好的描述：**
```
API 接口：
- GET /api/users - 获取用户列表（支持分页参数 page、size）
- POST /api/users - 创建用户（请求体：{ name, email }）
- PUT /api/users/:id - 更新用户
- DELETE /api/users/:id - 删除用户
```

**不好的描述：**
```
调用用户接口
```

## 📝 需求描述模板

### 完整需求模板

```
【功能名称】
用户管理

【功能描述】
完整的用户管理功能，包括列表展示、搜索、增删改查。

【页面结构】
1. 用户列表页面（/users）
   - 表格展示用户信息
   - 搜索栏（姓名、邮箱）
   - 操作按钮（添加、编辑、删除）
   - 分页组件

2. 添加/编辑用户页面（/users/add, /users/edit/:id）
   - 表单（姓名、邮箱、密码）
   - 提交按钮
   - 取消按钮

【API 接口】
- GET /api/users?page=1&size=10&name=xxx&email=xxx
- POST /api/users
- PUT /api/users/:id
- DELETE /api/users/:id

【技术要求】
- 使用 CommonTable 组件
- 使用 @/utils/request 发送请求
- 使用 Element Plus 组件
- 响应式设计

【参考】
- 参考 src/views/TableDemo.vue 的实现
- 参考 src/views/RequestDemo.vue 的 API 调用方式
```
